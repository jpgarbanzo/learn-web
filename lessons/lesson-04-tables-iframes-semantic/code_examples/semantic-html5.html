Document Structure Elements

These elements define the overall structure and organization of your document:

<header> Element

Purpose: Represents introductory content or navigational aids.

Can contain: Headings, logo, search form, author info

Usage: Page header, section header, or article header

<header>
  <h1>Site Title</h1>
  <p>Site tagline or description</p>
</header>

<nav> Element

Purpose: Contains navigation links to other pages or sections.

Best practices: Use for major navigation blocks, not all link groups

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
  </ul>
</nav>

<main> Element

Purpose: The main content of the document body.

Rules: Only one main element per page, excludes repeated content

Contains: The primary content unique to this page

<footer> Element

Purpose: Footer for nearest sectioning content or root.

Typical content: Author info, copyright, related links

<footer>
  <p>© 2024 Company Name. All rights reserved.</p>
  <nav><!-- Footer navigation --></nav>
</footer>

Content Sectioning Elements

These elements organize content into logical sections:

<section> Element

Purpose: Generic section of content with a thematic grouping.

Rule: Should have a heading (h1-h6) to describe the section

Use when: Content is related but doesn't warrant its own page

Example: Product Features Section

<section>
  <h2>Features</h2>
  <p>Our product offers these benefits...</p>
</section>

<article> Element

What is an Article?

Published: January 15, 2024

Purpose: Self-contained, reusable content that could stand alone.

Examples: Blog posts, news articles, product reviews, comments

Test: Would this content make sense in an RSS feed?

<article>
  <header>
    <h2>Article Title</h2>
    <p>By Author Name</p>
  </header>
  <p>Article content...</p>
</article>

This article demonstrates the article element structure.

<aside> Element

Purpose: Content tangentially related to main content.

Examples: Sidebars, pull quotes, advertisements, related links

Context matters: Can be used within articles or as page-level sidebar

"The aside element is perfect for content that supports but doesn't belong in the main flow."

Text-Level Semantic Elements

These elements add meaning to text content:

Date and Time Elements

Use the <time> element for dates and times:

  • Event date: December 25, 2024
  • Publication time: 2:30 PM on January 15
  • Duration: 2 hours 30 minutes
<time datetime="2024-12-25">Christmas Day</time>
<time datetime="2024-01-15T14:30:00">2:30 PM today</time>

Highlighting and Emphasis

Different ways to emphasize text content:

  • <mark>: Highlighted text (like a highlighter)
  • <strong>: Important text (strong importance)
  • <em>: Emphasized text (stress emphasis)
  • <small>: Fine print or side comments

Example: This quarter showed significant growth in sales, which was particularly impressive given market conditions. (Data subject to revision)

Frequently Asked Question

Q: How do semantic elements help with SEO?

A: Semantic elements help search engines understand your content structure, which can improve how your pages are indexed and displayed in search results.

Complete Semantic Structure Example

Here's how semantic elements work together in a typical webpage:

<!DOCTYPE html>
<html lang="en">
<head>
    <title>Blog Post Example</title>
</head>
<body>
    <header>
        <h1>My Blog</h1>
        <nav>
            <ul>
                <li><a href="/">Home</a></li>
                <li><a href="/about">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <header>
                <h1>Understanding Semantic HTML</h1>
                <p>Published <time datetime="2024-01-15">January 15, 2024</time></p>
            </header>
            
            <section>
                <h2>Introduction</h2>
                <p>Semantic HTML provides meaning...</p>
            </section>
            
            <section>
                <h2>Key Benefits</h2>
                <ul>
                    <li><strong>Accessibility</strong>: Screen readers understand structure</li>
                    <li><strong>SEO</strong>: Search engines parse content better</li>
                </ul>
            </section>
            
            <footer>
                <p>Tags: HTML, Semantics, Accessibility</p>
            </footer>
        </article>
        
        <aside>
            <h2>Related Articles</h2>
            <ul>
                <li><a href="/html-basics">HTML Basics</a></li>
                <li><a href="/css-intro">CSS Introduction</a></li>
            </ul>
        </aside>
    </main>
    
    <footer>
        <p>© 2024 My Blog. All rights reserved.</p>
    </footer>
</body>
</html>

Quick Reference

Semantic Elements Checklist
  • ✓ Use header for introductory content
  • ✓ Use nav for navigation links
  • ✓ Use main for primary content (only one per page)
  • ✓ Use section for thematic content groupings
  • ✓ Use article for standalone content
  • ✓ Use aside for tangential content
  • ✓ Use footer for closing content
  • ✓ Use time for dates and times
  • ✓ Use figure/figcaption for images and captions
Accessibility Benefits
  • Screen readers navigate by landmarks
  • Users can skip to main content
  • Content structure is clearly defined
  • Search and find functionality improved
SEO Benefits
  • Search engines understand content hierarchy
  • Featured snippets use semantic structure
  • Rich results leverage structured content
  • Mobile-first indexing relies on semantics

Remember: Choose elements based on meaning, not appearance. CSS handles the visual styling!
