❌ Inappropriate Use: Page Layout

Don't use tables for layout! Instead, use CSS Grid or Flexbox:


<!-- Wrong: Using table for layout -->
<table>
    <tr>
        <td>Header</td>
    </tr>
    <tr>
        <td>Sidebar</td>
        <td>Main Content</td>
    </tr>
</table>

<!-- Right: Using semantic elements with CSS -->
<header>Header</header>
<main>
    <aside>Sidebar</aside>
    <article>Main Content</article>
</main>

Table Accessibility Best Practices

Essential Elements

  • Caption: Always describe what the table contains
  • th elements: Use for headers, not just td with bold styling
  • scope attribute: Specify "col" or "row" to clarify header relationships
  • headers attribute: For complex tables, explicitly associate data with headers

Structure Guidelines

  • Use <thead>, <tbody>, and <tfoot> to group content
  • Keep table structure simple when possible
  • Provide summaries for complex data relationships
  • Test with screen readers and keyboard navigation

Responsive Considerations

  • Consider horizontal scrolling for wide tables on mobile
  • Use CSS to make tables more mobile-friendly
  • Provide alternative views for complex tables on small screens
  • Ensure touch targets are large enough for mobile users

Remember: Tables are for tabular data, not page layout. Always prioritize accessibility and semantic meaning!
