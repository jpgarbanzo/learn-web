Audio Attributes Explanation

controls
Displays play/pause button, volume control, and progress bar
autoplay
Automatically starts playing when page loads (often blocked by browsers)
loop
Repeats the audio file continuously
muted
Starts the audio muted (required for autoplay in most browsers)
preload
  • auto - Download entire file when page loads
  • metadata - Only download file info (duration, etc.)
  • none - Don't download anything until user plays

Multiple Sources and Browser Support

Why Multiple Formats?

Different browsers support different media formats. Providing multiple sources ensures maximum compatibility.

Common Audio Formats:

  • MP3 - Widely supported, good compression
  • OGG - Open source, good quality
  • WAV - Uncompressed, best quality but large files

Common Video Formats:

  • MP4 (H.264) - Most widely supported
  • WebM - Open source, smaller files
  • OGV - Open source alternative

Performance Best Practices

  • File Size: Optimize media files for web delivery
  • Preload Strategy: Use appropriate preload settings
    • preload="none" - For background or optional content
    • preload="metadata" - For content users might play
    • preload="auto" - For content users will likely play
  • Responsive Design: Use percentage widths and max-width
  • Loading States: Provide poster images and fallback content

User Experience Guidelines

  • Avoid Autoplay: Let users control when media plays
  • Provide Controls: Always include controls attribute
  • Consider Data Usage: Large files impact mobile users
  • Test Across Devices: Ensure compatibility on mobile and desktop
